# 00 · 产品定义

> 这份文档定义 ListPack Studio 是什么、给谁、解决什么、不做什么、按什么节奏发布。
> 所有决策依据见 [`docs/research/`](../research/00-executive-summary.md)，本文不重复论证。

---

## 1. 一句话定位

> **Upload one photo. Pass Amazon, Shopify, Temu, SHEIN review the first time. Get the full listing pack + ad creative refreshed every 10 days. From $49/month.**

中文版（落地页备用）：

> 上传一张商品图，**一次过审**亚马逊 / Shopify / Temu / SHEIN，自动产出整套上架素材，每 10 天自动刷新广告创意。$49/月起。

## 2. 价值主张（按重要性排序）

| # | 价值 | 替代方案 vs ListPack | 用户感知 |
|---|------|----------------------|----------|
| 1 | **一次过审** | Photoshop 反复试 + 被拒 + 申诉 = 1-7 天 | "省 3 次返工" |
| 2 | **多平台原生** | Canva 手动切 5 套尺寸 = 半天/SKU | "省半天劳动" |
| 3 | **真 Agent 自动化** | 选模板 → 出图 → 评估 → 改 → 再出 (人工 30 min) | "AI 帮我跑完整链" |
| 4 | **持续素材引擎** | UGC 创作者 $300-$2K/条 + 自己剪 7-10 天换 | "广告永不断粮" |
| 5 | **品牌可沉淀**（v2） | Penji $499/月 + 反复对齐品牌 brief | "AI 记住我的品牌" |

**反向价值（话术不要说）**：
- ❌ "自动加 AI 披露水印" → 改写："**通过审核保证**"（用户讨厌主动声明 AI，参考 [04-user-pain-points-jtbd § D4](../research/04-user-pain-points-jtbd.md)）
- ❌ "AI 虚拟模特" → 头部都做不好，v1 不承诺
- ❌ "完整短视频生成" → v1 只承诺"2-3 秒商品旋转动图"
- ❌ "替代你的设计师" → 改写："**让你的设计师省 80% 重复劳动**"（DTC 段最大反对意见是"我们有内部设计师"）

## 3. ICP（Ideal Customer Profile）

### 3.1 三类客户

| 维度 | A 个人卖家 | **B DTC（核心 ICP）** ★ | C 代运营/MCN |
|---|---|---|---|
| SKU 规模 | <100 | **100-2000** | 多品牌 × 多 SKU |
| 团队 | 1-3 人 | **3-20 人** | 3-50 人 |
| 当前月支出 | $20-50 | **$1K-5K** | $500-3K/客户 |
| 月付费甜蜜点 | $9 / $19 | **$49 / $99 / $199** | $499 / $1499 |
| 月流失率（行业基准） | 8-10% | **3-5%** | 2-3% |
| LTV/CAC | 2-4x | **5-10x** | 10-30x |
| 决策周期 | 5 分钟 | **2 周** | 1-2 个月 |
| 决策者 | 自己 | **创始人 + 创意负责人 + 设计师** | 老板 + 运营总监 + 财务 |
| 主入口 | App Store + YouTube | **Shopify App Store** + DTC Newsletter | AMZ123 + 行业会展 + Partner |
| 最痛 | 主图被打回 + 抠图崩 | **多平台尺寸 + A+ 速度 + 素材消耗** | 多品牌调性 + 客户改稿 |

### 3.2 冷启动顺序 = **B → A → C**

```
0-6 月    B：DTC（核心 ICP，CAC/LTV 健康）
6-12 月   A：个人卖家做 freemium 漏斗
12 月+    C：销售驱动开发 Agency / 白标
```

**不能反过来**——C 段销售周期 1-2 个月，没有 B 段案例时 Agency 不会试。

### 3.3 红线客户 / 红线品类（明确不做）

- **客户**：年入 <$5K 的爱好者（CAC 收不回）；>$50M GMV 大企业（销售周期 6-12 个月）；强品牌调性奢侈品（创意必须手工）
- **品类**：保健品 / 医美 / 珠宝 / 食品（监管风险 + AI 出图退货率最高）
- **平台**：v1 不做中国电商（淘宝/拼多多/抖音）—— 跨境优先

## 4. 功能清单（v1 / v2 / v3 三阶段）

### 4.1 v1 MVP（90 天，B 段付费可用）

| 优先级 | 功能模块 | 一句话描述 | 验收信号 |
|---|---|---|---|
| P0 | **Amazon 主图合规检查** | 上传图 → 自动检测白底/85% 占比/文字/水印/道具 → 出 pass/fail 报告 + 一键修复 | 100 张测试图，pass-first-time ≥ 90% |
| P0 | **多平台尺寸自动适配** | 一张源图 → 同时输出 Amazon / Shopify / eBay / Temu / SHEIN 主图+副图全套尺寸 | 5 个平台 × 主图+3 张副图 = 20 张/SKU 一次完成 |
| P0 | **场景图生成（lifestyle）** | 自然语言描述场景 → 商品保真 + 背景换 → 输出 3-5 张选择 | 单 SKU 3 分钟内出 5 张可用 lifestyle |
| P0 | **A+ Content 模块化生成** | 7 个固定模块（hero / 功能 / 对比 / 详情）→ 用户选模块 + 填卖点 → 输出 970×600 合规图 | Amazon Brand Registry 卖家可直接上传 |
| P0 | **真 Agent 模式（多步）** | 输入"做亚马逊上架包" → 自动规划 → 出主图 → VLM 评估 → 改 → 出副图 → 出 A+ → 打包 | 用户全程"watch agent 做事"，可暂停/分支 |
| P0 | **合规元数据自动写入**（话术：Safety Layer） | 输出图自动嵌 C2PA + EU AI Act 兼容 metadata（用户不可见） | 100% 输出图通过 C2PA Verify |
| P1 | **品类规则引擎 v1** | 用户选品类（服装/家居/3C/配饰/玩具）→ 自动套用违禁词 / 必备元素规则 | 5 个品类规则上线 |
| P1 | **被拒原因可解释报告** | 检测失败时给出"红线条款引用 + 修复建议" | 每条 fail 必带规则编号 + 修复 CTA |
| P1 | **Shopify 一键直发** | 出图后 → push 到 Shopify Product / Variant | Shopify App Store 上架，安装即用 |
| P2 | **商品旋转动图** | 单图 → 2-3 秒商品 360°/zoom 动图（非完整视频） | 输出 GIF / MP4 |

**v1 明确不做**（写进 PRD 是为了挡住后续诱惑）：
- ❌ AI 模特 / 虚拟试穿
- ❌ 完整短视频生成（>5 秒）
- ❌ LoRA / 品牌自训
- ❌ 团队协作 / 多人画布
- ❌ 中国电商平台支持
- ❌ 移动端 App（Web 优先）

### 4.2 v2（6-12 个月，B 高级 + 留存壁垒）

| 优先级 | 功能模块 | 一句话描述 |
|---|---|---|
| P0 | **Brand Kit / 品牌资产库** | 上传 30-50 张品牌图 → 提取品牌色 / 字体 / 版式 / 模特特征 → 后续生成强制对齐 |
| P0 | **轻量 LoRA 训练** | 5-20 张品牌图 → 30 分钟自训 → 生图保持品牌一致性 |
| P0 | **持续投放素材引擎** | 每 7-10 天自动产出新一轮 Meta/TikTok 创意 + 历史素材表现回流 |
| P1 | **团队协作 + 评审流** | 3 席团队 / @ 评论 / 审批通过后才能下载 |
| P1 | **Amazon 一键直发** | 通过 Amazon SP-API push 主图 + A+ |
| P1 | **AI 模特 v1** | 服装垂类切入（家居 / 3C 不做模特） |
| P2 | **品类规则引擎 v2** | 扩到 10 个品类 + 多语言违禁词 |

### 4.3 v3（12 个月+，C Agency / API）

| 优先级 | 功能模块 | 一句话描述 |
|---|---|---|
| P0 | **Agency Workspace** | 多客户工作区隔离 / 客户分账 / 白标 |

## 5. 商业模式

### 5.1 价格分层（Hybrid：订阅为主 + 超额按量 + Enterprise/API 按量）

| 档位 | 价格 | 配额（SKU/月） | 超额费率 | 卖给谁 | 关键功能 |
|---|---|---|---|---|---|
| **Free** | $0 | 5，含水印 | 不允许超额 | A 漏斗 | 主图合规检查 + 1 个场景图 |
| **Starter** | $19/mo | 30，无水印 | $0.80 / SKU | A 付费转化 | + 多平台尺寸 + A+ 简版 |
| **Pro** ⭐ | **$49/mo** | **100** + 详情长图 + 一次过审保证 | **$0.50 / SKU** | **B 入门（核心档）** | + 真 Agent 多步规划 |
| **Brand** | $149/mo | 500 + Brand Kit + Shopify 直连 | $0.30 / SKU | B 高级 | + 品牌资产库（v2）+ 团队 3 席 |
| **Agency** | $499/mo | 5 客户 workspace + API + 白标 | $0.20 / SKU | C | + 客户隔离 + 月度审计报告 |
| **Enterprise** | 定制 | API 量 / SLA / 合规承诺函 | 谈定 | 大型 DTC / Agency | + 私有 LoRA / SSO / 合同期 |
| **Developer API**（v3） | 纯按量 | — | $0.10-0.30 / 张（按操作类型） | 开发者 / 后端集成 | API key + 月度账单 |

**超额费率设计逻辑**（超得再多都不如升档）：
- Starter 超 38 张 = $30 = 直接升 Pro 更划算（多 70 SKU）
- Pro 超 200 张 = $100 = 直接升 Brand 更划算（多 400 SKU）
- Brand 超 1167 张 = $350 = 直接升 Agency 更划算



### 5.2 三条永不让步的底线（参考 Photoroom 1.3 星反面教材）

1. **定价透明**——不藏配额、不偷涨价、年付不强制
2. **不偷下线已付费功能**——付费功能下线必须 90 天提前通知 + 退款选项
3. **不暗扣费**——试用期到期前 48h 必须主动邮件 + 一键退订

写进用户协议 + 落地页 footer。

### 5.3 Refund / 试用规则

- **Free**：永久免费，无 credit card
- **Starter / Pro**：7 天全额退款 + 试用期不收 card（参考 Booth.ai 死因 § 2）
- **Brand / Agency**：14 天全额退款
- **Enterprise**：合同条款定制

### 5.4 Overage Policy（超额规则，3 条硬约束）

为什么需要：B 段卖家有季节性（双 11/黑五/Prime Day），临时溢出 50-100 SKU 不应该被迫升档；但也不能让账单失控。

| 规则 | 约束 |
|---|---|
| **1. 月度超额上限** | 每月超额不得超过订阅配额的 **50%**（例：Pro 100 SKU 月最多超 50 张 = $25 额外）。超过 cap 后**不停服**，自动降级生成质量（参考 v1 → 模板版，不调用 Agent），保留体验 |
| **2. 提前通知** | 用量达 80% / 100% / 触发 overage 时各发一次邮件 + 站内提醒（参考 Vercel / OpenAI 的最佳实践） |
| **3. 透明账单** | 每张超额 SKU 单独列在月度账单，可下载明细。任何时刻可在设置里关闭 overage（关掉后用完即停） |

### 5.5 为什么不做纯 Credit 制（重要决策记录）

参考 [05-competitor-matrix § Booth.ai 死因 #2](../research/05-competitor-matrix.md) 和 [02-customer-segments § B.4 反对意见](../research/02-customer-segments.md)：

- **Booth.ai** 用 credit-heavy + pay-first，用户在感受到价值前就被摩擦掉 → 死
- **SellerPic** credit 制被 Trustpilot 抱怨"不透明"
- **Uwear.ai** 是唯一做对纯按量的（$0.10/credit + 永不过期），但目标客户是开发者不是 B 段卖家
- **B 段 DTC** 最大需求是"月度预算可预测"，纯按量无法满足

→ 主线必须订阅。API（v3）才可纯按量。

## 6. 北极星指标 + 健康指标

### 6.1 北极星指标（一个就够）

**Listing-Pack-Generated-and-Used / 月**
（"用户生成并实际下载/发布出去的完整上架包数量"——不是单图数）

理由：单图生成量会被用户测试行为污染（生成 100 张选 1 张）；只有"下载/发布"才代表真实价值实现。

### 6.2 健康指标（产品健康 / 商业健康 / 信任）

| 维度 | 指标 | v1 6 个月目标 |
|---|---|---|
| 产品 | Pass-First-Time Rate（合规检查通过率） | ≥ 90% |
| 产品 | Agent 完整跑完率（中途不被人工打断） | ≥ 70% |
| 产品 | 平均"上传图 → 拿到完整包"时长 | ≤ 5 分钟 |
| 商业 | Pro 付费用户数 | 100 个 |
| 商业 | MRR | $5K |
| 商业 | 月流失率（Pro 段） | ≤ 5% |
| 信任 | Trustpilot 评分 | ≥ 4.0（行业是 1.3-1.4，超 4 即护城河） |
| 信任 | 7 天试用 → 付费转化率 | ≥ 15% |

## 7. 关键非目标 / 取舍说明

| 别人做 | 我们不做 / 暂不做 | 原因 |
|---|---|---|
| Photoroom：移动端 App | v1 不做 App | B 段 DTC 都在桌面/Shopify Admin，移动 App 是 A 段才需要 |
| Pebblely：API 公开 | v1 不开 API | API 是 v3 Agency 才有的功能，过早开会被白嫖 |
| Flair.ai：节点画布 | 任何阶段都不做节点画布 | 用户是美工不是工程师，节点是反向选择 |
| Canva：通用设计 | 不做平面设计/海报/PPT | 聚焦电商场景，宽度=死亡 |
| SellerPic：AI 模特 + 视频 | v1 不承诺，v2 才做服装模特 | 头部都做不好，承诺会伤口碑 |
| Ecomtent：Amazon SEO 优化 | 不做 SEO / 文案 / Listing 文字优化 | 焦点是图，扩到文案战线太长 |

## 8. v1 30-45 天加速里程碑（AI 开发模式）

详细工程实施见 [`04-open-source-stack.md`](04-open-source-stack.md)。

**前提**：代码由 AI 写、单人决策、无团队评审/沟通成本。市场验证（Phase 0 4 周）**独立于代码节奏**并行跑。

| 天 | 里程碑 | 关键交付 |
|---|---|---|
| D1-3 | 选型 + 仓库初始化 + CI/CD + 域名 + 落地页骨架 | repo 跑得起来，preview 自动部署 |
| D4-10 | 合规规则引擎 v1（Amazon 主图 10 条规则 + Temu/SHEIN/eBay/Shopify 基础规则） | `/v1/compliance/check` 可用，命中率 ≥ 90% |
| D11-18 | 多平台尺寸适配 + 场景图生成 + A+ Content 模块化 | 单图 → 全平台 20 张图能跑通 |
| D19-25 | Agent 编排（Painter-Commenter 闭环 + 多步规划 + SSE 流式输出） | 完整 listing pack 端到端跑通 |
| D26-30 | Shopify App 提交 + 计费系统 + Landing 上线 | App Store 待审、Stripe 接通、用户能注册付费 |
| D31-45 | **缓冲期**：Bug 修复 / 性能优化 / 5 个种子客户试用反馈 / Pro 档开放付费 | 第 1 个付费用户、Trustpilot 第一条好评 |

**v2/v3 何时启动**——不按时间表，按市场信号：
- v2 启动条件：v1 月流失率 ≤ 5% 且 Pro 付费用户 ≥ 50
- v3 启动条件：v2 上线后至少 1 个 Agency / Enterprise 客户主动来询

**不建议无视市场信号直接做 v2/v3**——见本节开头"为什么保留阶段"。
